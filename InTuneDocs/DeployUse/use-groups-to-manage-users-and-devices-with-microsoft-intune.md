# Использование групп для управления пользователями и устройствами в Microsoft Intune

В этой статье описывается создание групп в Intune. Здесь также содержатся сведения о том, какие изменения в управлении группами произойдут в ближайшие месяцы. Дополнительные сведения о *текущем* подходе к управлению группами см. в разделе [Создание групп для управления пользователями и устройствами в Microsoft Intune](#Create-groups-to-manage-users-and-devices-with-Microsoft-Intune) этой статьи.

## Уведомление о предстоящих улучшениях в среде администрирования групп

На основе ваших отзывов о том, что необходимо единообразие среды выбора целей и работы с группами в компонентах Enterprise Mobility и Enterprise Security, мы преобразовали группы Intune в группы безопасности на основе Azure Active Directory. Это позволит унифицировать управление группами в Intune и Azure Active Directory (Azure AD). Новая среда работы позволяет отказаться от дублирования групп между службами и обеспечивает расширяемость с помощью PowerShell и Graph. 

### Как это повлияет на работу?
Это изменение пока не влияет на вашу работу, но мы расскажем вам, чего следует ожидать.

-   В сентябре для новых учетных записей, подготовленных после ежемесячного выпуска службы, будут использоваться группы безопасности Azure AD, а не группы пользователей Intune.   
-   В октябре новые учетные записи, подготовленные после ежемесячного выпуска службы, будут управлять группами пользователей и устройств на портале Azure AD. Это изменение не повлияет на существующих клиентов.
-   В ноябре группа разработчиков Intune начнет миграцию существующих клиентов в новую среду управления группами на основе Azure AD. Все текущие группы пользователей и устройств Intune будут перенесены в группы безопасности Azure AD. Миграция будет выполняться поэтапно начиная с ноября. Мы не приступим к миграции, пока не будем уверены, что сможем свести к минимуму влияние на вашу повседневную деятельность и обеспечить бесперебойную работу пользователей. Кроме того, перед миграцией вашей учетной записи вы получите уведомление от корпорации Майкрософт.


### Как и когда будет выполняться переход на новые группы?
Миграция текущих клиентов будет выполняться постепенно. Мы составим расписание миграции и обновим этот раздел через несколько недель, чтобы предоставить вам дополнительные сведения. Вы получите предварительное уведомление о миграции. Если при миграции у вас возникнут проблемы, обратитесь к рабочей группе по миграции по адресу [intunegrps@microsoft.com](intunegrps@microsoft.com).

### Что произойдет с существующими группами пользователей и устройств?
 Все созданные вами группы пользователей и устройств будут перенесены в группы безопасности Azure AD. Группы Intune по умолчанию, например "Все пользователи", будут перенесены, только если вы используете их в развертываниях на момент миграции. Миграция может быть более сложной для некоторых групп, и мы уведомим вас, если потребуются какие-либо дополнительные действия.

### Какие новые возможности будут доступны?
Будут доступны новые функции, перечисленные ниже.

-    Группы безопасности Azure AD будут поддерживаться в Intune для всех типов развертывания.
-    Группы безопасности Azure AD будут поддерживать не только группы пользователей, но и устройств.
-    Группы безопасности Azure AD будут поддерживать динамические группы с атрибутами устройств Intune. Например, вы сможете динамически группировать устройства на базе платформы, например iOS. Таким образом, при регистрации нового устройства iOS в вашей организации оно будет автоматически добавляться в динамическую группу устройств iOS.
-    Возможности совместного администрирования для управления группами Azure AD и Intune.
- *Роль администратора службы Intune* будет добавлена в Azure AD, чтобы разрешить администраторам служб в Intune выполнять задачи управления группами в Azure AD.




### Какие функции Intune не будут доступны?
Работа с группами станет удобнее, но после миграции некоторые функции Intune будут недоступны.

#### Функции управления группами

-   Вы не сможете исключать членов и группы при создании новой группы. Тем не менее динамические группы Azure AD позволяют использовать атрибуты для создания расширенных правил, с помощью которых можно исключать членов на основе условий.
-   Будет прекращена поддержка групп **Несгруппированные пользователи** и **Несгруппированные устройства**. Эти группы переноситься не будут.


#### Функции, зависящие от групп

-   Роль администратора служб утратит разрешения на **управление группами**.
-   Вы не сможете группировать устройства Exchange ActiveSync.  Группа **Все управляемые устройства EAS** будет преобразована из представления группы в представление отчета.
-  Сведение с группами в отчетах будет недоступно.
-  Настраиваемое нацеливание правил уведомлений для групп будет недоступно.

### Как можно подготовиться к этому изменению?
 Ниже приведены рекомендации, которые помогут вам упростить переход.

- Удалите все нежелательные или ненужные группы Intune перед миграцией.
- Оцените использование режима исключения в группах и рассмотрите возможность преобразования групп таким образом, чтобы отказаться от исключений.
-  Если в организации есть администраторы, у которых нет разрешений на создание групп в Azure AD, попросите администратора Azure AD добавить их в роль Azure AD **Администратор служб Intune**.


## Создание групп для управления пользователями и устройствами в Microsoft Intune

В этом разделе описывается создание групп Intune в консоли администрирования Intune.

Для создания групп и управления ими используется рабочая область **Группы** в консоли администрирования Microsoft Intune. На странице **Обзор групп** содержатся сводные данные о состоянии, позволяющие выявлять проблемы и определять их приоритеты в следующих направлениях.

-   Предупреждения
-   Обновления программного обеспечения
-   Endpoint Protection
-   Политика
-   Управление программным обеспечением

Кроме того, иерархия групп отображается вместе со сводками состояния, помогающими выявлять и устранять проблемы для членов выбранной группы.


> [!TIP]
> При создании групп учитывайте, как именно планируется применить политику. Например, можно использовать политики для конкретных операционных систем устройств и политики для различных ролей в организации или для подразделений, уже определенных в Active Directory. Некоторые считают, что удобно иметь отдельные группы устройств для iOS, Android и Windows, а также группы пользователей для каждой роли в организации.
>
> Возможно, вам потребуется создать политику по умолчанию, которая применяется ко всем группам и устройствам, чтобы обеспечить соответствие некоторым основным требованиям вашей компании. После этого можно создать более конкретные политики для наиболее обширных категорий пользователей и устройств, например политики электронной почты для каждой операционной системы устройств.
>
> Ответственно отнеситесь к именованию политик, чтобы их можно было легко найти в последствии. Например, хорошим и содержательным именем политики является **Политика электронной почты WP для всей компании**.
>
> Каждый раз при создании ограничительной политики потребуется донести ее смысл до пользователей, поэтому после создания более общих групп и политик уделите особое внимание созданию групп меньшего размера, чтобы исключить излишнее взаимодействие.


## Создание группы устройств

1.  В консоли администрирования Intune выберите **Группы** &gt; **Обзор** &gt; **Создать группу**.

2.  Укажите имя и необязательное описание группы и выберите группу устройств в качестве родительской группы. Нажмите кнопку **Далее**.

3.  На странице **Определение условий членства** выберите тип устройств, которые будут входить в группу. Дополнительные параметры для настройки группы зависят от выбранного типа устройств.

    -   **Компьютер:** укажите, следует ли включать всех членов родительской группы, а также включаемые или исключаемые подразделения и домены. Сведения о подразделениях и доменах компьютера см. в инвентаризационных данных

    -   **Мобильные:** укажите включение только мобильных устройств, управляемых Intune и/или Exchange ActiveSync.

    -   **Все устройства:** этот параметр включает все устройства без исключений на основании условий.

4.  На странице **Определение прямого членства** включите или исключите отдельные устройства, нажав кнопку **Обзор**. Если этот параметр используется для выбора устройств, не входящих в указанную родительскую группу, эти устройства автоматически добавляются в родительскую группу.


5.  На странице **Сводка** проверьте действия, которые будут выполнены. Нажмите кнопку **Готово**.

Созданная группа появится в списке **Группы** в рабочей области **Группы** в родительской группе. Здесь также можно изменять или удалять группу.

## Создание группы пользователей

1.  В консоли администрирования Intune выберите **Группы** &gt; **Обзор** &gt; **Создать группу**.

2.  Укажите имя и необязательное описание группы и выберите группу пользователей в качестве родительской группы. Нажмите кнопку **Далее**.

3.  На странице **Определение условий членства** укажите, следует ли включать всех членов родительской группы или нужно начать с пустой группы.  Вы можете включать или исключать членов на основе **групп безопасности** пользователей, вручную настраиваемых в [Центре администрирования Office 365](http://go.microsoft.com/fwlink/?LinkId=698854) или синхронизируемых из локальной службы Active Directory. При изменении членства в группе безопасности происходит изменение членства в группах пользователей, основанных на этой группе безопасности.

    > [!IMPORTANT]
    > В настоящее время если ваша группа содержит члены из определенных групп безопасности или групп руководителей и вы исключаете членов из определенных групп, изначально включенные вами члены будут удалены. Чтобы создать группу, которая содержит как включенные, так и исключенные члены, рекомендуется сначала создать родительскую группу с включенными членами, а затем создать дочерний элемент для этой группы, в котором перечислены исключенные члены. Затем эту дочернюю группу можно использовать для политик Intune, профилей и распространения приложений.

    > [!NOTE]
    > На портале управления Azure можно создать группу на основе того руководителя, которому подотчетны пользователи. Эта группа будет динамической и станет изменяться по мере добавления или удаления сотрудников в команде этого руководителя в Azure Active Directory. Процедура создания группы Azure на основе руководителя описана в разделе **Настройка группы в качестве группы "Руководитель"** статьи [Использование атрибутов для создания расширенных правил](https://azure.microsoft.com/en-us/documentation/articles/active-directory-accessmanagement-groups-with-advanced-rules/).


4.  На странице **Определение прямого членства** включите или исключите отдельных пользователей, нажав кнопку **Обзор**. Если этот параметр используется для выбора пользователей, не входящих в указанную родительскую группу, эти пользователи автоматически добавляются в родительскую группу. В нижней части диалогового окна **Выбор членов** находится параметр для ручного добавления пользователя. Это удобно, если вы хотите добавить пользователя, у которого пока нет зарегистрированного устройства.


5.  На странице **Сводка** проверьте действия, которые будут выполнены. Нажмите кнопку **Готово**.

Созданная группа появится в списке **Группы** в рабочей области **Группы** в родительской группе. Здесь также можно изменять или удалять группу.

> [!TIP]
> Группы безопасности отлично подходят для заполнения групп пользователей. Так как группы безопасности определяют, кто именно имеет доступ к определенным ресурсам, эти сведения можно легко преобразовать в группы пользователей Intune. Все группы безопасности, синхронизируемые из Active Directory с Azure Active Directory или созданные непосредственно в Azure Active Directory с помощью Центра администрирования Office 365 или портала администрирования Azure, доступны для создания групп пользователей в Intune.

## Настройка представлений для ролей администратора
Отфильтрованные представления групп позволяют настраивать представление, видимое администраторам, на основе их роли и ограничить набор групп, которыми может управлять каждый ИТ-администратор. Это может быть полезно в следующих случаях.

-   ИТ-администраторы должны иметь возможность развернуть элементы только для определенных пользователей и устройств.

-   Для каждого ИТ-администратора нужно отобразить только соответствующие группы.

Отфильтрованные представления групп можно настроить для администраторов службы на консоли администрирования Intune. Дополнительные сведения см. в статье [Предварительные сведения перед началом работы с Microsoft Intune](/intune/get-started/what-to-know-before-you-start-microsoft-intune).

После настройки отфильтрованных представлений групп для администратора службы этот администратор может выполнять следующие действия:

-   Просматривать и выбирать только группы, указанные при развертывании программного обеспечения или политик, а также при просмотре отчетов.

-   Не получает сведения о состоянии на следующих страницах консоли администрирования:

    -   **Обзор системы**

    -   **Общие сведения о группах**

    -   **Обзор Endpoint Protection**

    -   **Общие сведения об оповещениях**

    -   **Обзор программного обеспечения**

    -   **Общие сведения о политике**

### Настройка отфильтрованных представлений групп

1.  В консоли администрирования Intune выберите **Администрирование** &gt; **Управление администраторами** &gt; **Администраторы служб**.

2.  Выберите администратора службы, для которого требуется отфильтровать группы и нажмите кнопку **Управление группами**.

3.  В диалоговом окне **Выбор групп, которые будут отображаться администраторам служб** добавьте группы, доступ к которым может получить выбранный администратор службы, а затем нажмите кнопку **ОК**.

После настройки отфильтрованных представлений групп ИТ-администратор сможет просмотреть и выбрать только выбранные вами группы.

## Управление группами
После создания групп ими можно управлять в соответствии с потребностями организации.

Вы можете изменить имя и описание группы, а также ее членов.

Вы можете удалить группу, если она больше не удовлетворяет потребностям вашей организации. При удалении группы пользователи, которые входят в нее, не удаляются.

## Дальнейшие действия

### Проверка структуры
После настройки групп и политик проверьте практическую значимость такой структуры, изучив **Предполагаемое значение** и **Состояние**.

1. Выберите любое устройство в группе устройств и просмотрите категории сведений в верхней части экрана.
2. Выберите элемент **Политика**. Вы увидите нечто похожее на этот снимок экрана с параметрами политик устройства Android.

![Пример политики параметров iOS](../media/Intune-Device-Policy-v.2.jpg)

Каждая политика имеет **Предполагаемое значение** и **Состояние**. Предполагаемое значение — это то, чего вы намереваетесь достичь при назначении политики. Состояние — это реальный совокупный результат всех примененных к устройству политик, а также требований и ограничений оборудования и операционной системы.  На снимке экрана можно увидеть два понятных примера:

-   Для параметра **Разрешить использование простых паролей** задано значение **Да**, как показано в столбце **Предполагаемое значение**, однако он имеет **Состояние** **Неприменимо**. Это вызвано тем, что простые пароли не поддерживаются для устройств Android.

-   Аналогично, элемент расширенной политики **Параметры электронной почты для устройств iOS** не применяется к этому устройству, так как оно работает под управлением Android.

> [!NOTE]
> Помните, что при наличии двух политик с разными уровнями ограничений, применимых к одному и тому же устройству или пользователю, используется более строгая политика.
